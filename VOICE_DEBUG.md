# 🎤 음성 인식 문제 해결 체크리스트

## 증상: 음성 인식 후 아무 반응이 없음

### 1단계: 브라우저 개발자 도구로 확인

iPhone Safari에서:
1. Safari → 설정(톱니바퀴) → 고급 → Web Inspector 활성화
2. Mac Safari에서: 개발 → [iPhone 이름] → [페이지 선택]
3. Console 탭에서 다음 로그 확인:
   - "음성 인식 결과: ..."
   - "검색 시작: ..."
   - "추출된 물건 이름: ..."
   - "검색 결과: ..."

### 2단계: 등록된 물건 이름 확인

**중요**: 음성으로 말한 이름과 등록된 이름이 정확히 일치해야 합니다!

예시:
- 등록: "잼" → 검색: "잼 어디있어?" ✅
- 등록: "딸기잼" → 검색: "잼 어디있어?" ❌
- 등록: "리모컨" → 검색: "리모콘 어디있어?" ❌ (발음 차이)

**팁**: 등록할 때 음성 인식이 잘 인식하는 이름으로 등록하세요.

### 3단계: 지원되는 질문 형식

다음 형식들이 인식됩니다:
- "잼 어디있어?"
- "잼 어디야?"
- "잼 어디에 있어?"
- "잼 어디?"
- "잼 찾아줘"
- "리모컨이 어디있어?"
- "열쇠는 어디있어?"

### 4단계: 테스트 순서

1. **먼저 간단한 물건 등록**
   - 이름: 잼
   - 위치: 찬장

2. **음성 검색 버튼 클릭**

3. **마이크 아이콘이 빨간색으로 변하는지 확인**
   - 변하지 않으면 → 마이크 권한 문제
   - 변하면 → 다음 단계 진행

4. **명확하게 말하기**
   - "잼 어디있어?"
   - (너무 빠르거나 느리지 않게)

5. **"들은 내용" 확인**
   - 화면에 "들은 내용: 잼 어디있어?" 표시되는지 확인
   - 표시되지 않으면 → 음성 인식 실패
   - 표시되면 → 다음 단계 진행

6. **결과 메시지 확인**
   - 성공: "잼은(는) 찬장에 있습니다" (초록색)
   - 실패: "잼을(를) 찾을 수 없습니다" (파란색)

### 5단계: 흔한 문제들

#### 문제 1: 음성 인식은 되는데 "찾을 수 없습니다"
**원인**: 등록된 이름과 말한 이름이 다름

**해결**:
```
등록된 물건 목록을 확인하고,
음성으로 말한 것과 똑같이 등록되어 있는지 확인
```

#### 문제 2: 마이크 아이콘이 빨간색으로 안 변함
**원인**: 마이크 권한 없음 또는 HTTPS 아님

**해결**:
```
1. https:// 로 시작하는지 확인 (http:// 는 안 됨)
2. iPhone 설정 → Safari → 마이크 → 허용
3. 페이지 새로고침 후 다시 시도
```

#### 문제 3: "들은 내용"이 표시되지 않음
**원인**: 음성 인식 실패

**해결**:
```
1. 주변 소음 확인
2. 더 크고 명확하게 말하기
3. 인터넷 연결 확인 (음성 인식은 서버 처리)
```

#### 문제 4: 음성 응답이 안 들림
**원인**: iPhone 무음 모드 또는 볼륨 0

**해결**:
```
1. iPhone 측면 스위치 확인 (무음 모드 해제)
2. 볼륨 버튼으로 소리 키우기
```

### 6단계: 직접 테스트

콘솔에서 직접 테스트:
```javascript
// Safari 개발자 도구 콘솔에서
searchItem("잼 어디있어?");
```

### 디버깅 팁

콘솔에서 확인할 내용:
1. 음성 인식 결과가 올바른지
2. 추출된 물건 이름이 올바른지
3. 검색 API 응답이 올바른지
4. 음성 출력이 시작되는지

### 여전히 안 된다면

1. 서버 로그 확인 (라즈베리파이 터미널)
2. 브라우저 캐시 삭제
3. Safari 완전히 종료 후 재실행
4. 서버 재시작: `Ctrl+C` → `npm start`
5. iPhone 재부팅
